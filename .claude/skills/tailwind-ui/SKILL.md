# Tailwind UI

Use when building or styling UI.

## When to Use
- Creating UI components or pages
- Enforcing a consistent design system
- Mobile-first responsive layouts

## Principles
1. **Mobile-first** — design for small screens, scale up with breakpoints.
2. **Theme tokens** — colors/spacing/typography from a central theme, not magic values.
3. **Lucide icons** — consistent icon set.
4. **Accessible** — semantic HTML, focus states, labels, contrast.
5. **Composable** — small reusable components over one-off markup.

## Checklist
- [ ] Works at mobile, tablet, desktop breakpoints
- [ ] Uses theme tokens (no hardcoded hex unless intentional)
- [ ] Keyboard-navigable + visible focus
- [ ] No layout shift on load
